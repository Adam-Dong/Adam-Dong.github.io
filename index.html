什么是测试？

在编写测试之前，首先需要知道：你应该测试什么？如果你的目标是修改一个已有的 App，你首先应该对准备修改的组件编写测试。
通常，测试应当包含：

  • 核心功能：模型类和方法，以及它们和控制器的交互
  • 最常用的 UI 操作
  • 边际条件
  • bug 修复

FIRST 原则：测试的最佳实践

FIRST 是几个单词的缩写，简要描述了有效的单元测试需要什么条件，这些条件包括：

  1. Fast：测试的运行速度要快，这样人们就不介意你运行它们了。
  2. Independent/Isolated：一个测试不应当依赖于另一个测试。
  3. Repeatable：同一个测试，每次都应当获得相同的结果。外部数据提供者和并发问题会导致间歇性的出错。
  4. Self-validating：测试应当是完全自动化的，输出结果要么是 pass 要么是 fail，而不是依靠程序员对日志文件的解释。
  5. Timely：理想情况下，测试的编写，应当在编写要测试的产品代码之前。(先于产品代码)

遵循 FIRST 原则会让你的测试清晰和有用，而不会成为 App 的渊薮。

单元测试基本操作

  • 创建Unit Test Target

  • 编写测试代码 （SUT：被测系统 System Under Test）

    1. 在 setup() 中创建 SUT，在 tearDown() 中释放 SUT
    2. 测试方法的名字总是以 test 开头，后面加上一个对测试内容的描述。

使用 hexo 免费搭建个人博客

发表于 2017-12-11
今天尝试用 hexo + github 搭建一个个人博客，制作简单，主题丰富，而且是免费的，如果有兴趣的不妨试一试！

什么是 Hexo ？
Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

Markdown 又是什么呢？
Markdown 是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式。
有点类似 HTML 都是一种标记语言,就像是把我们平时使用word中的段落,字体大小,超链接等抽象成了固定的符号

比如: 在 word 中的添加超链接 需要选中文字后点击右键，找到超链接选项，再添加一个网址就可以了
在Markdown中，你只需要 [百度] (www.baidu.com) 就可以指定 百度这个两个字为一个超链接

下面来说说如何安装 Hexo

安装 Hexo 分为以下三步
* 安装 Git
* 安装 Node.js
* 安装 Hexo
安装前提

安装 Hexo 相当简单。然而在安装前，您必须检查电脑中是否已安装下列应用程序：

  • Node.js
  • Git

如果您的电脑中已经安装上述必备程序，那么恭喜您！接下来只需要使用 npm 即可完成 Hexo 的安装。

$ npm install -g hexo-cli

如果您的电脑中尚未安装所需要的程序，请根据以下安装指示完成安装。

安装 Git
  • Windows：下载并安装 git.
  • Mac：使用 Homebrew, MacPorts ：brew install git;或下载 安装程序 安装。
  • Linux (Ubuntu, Debian)：sudo apt-get install git-core
  • Linux (Fedora, Red Hat, CentOS)：sudo yum install git-core

Mac 一般自带git 可以通过在 Terminal 中输入 git version来查看版本号，如果返回版本号说明已经安装，显示 command not found 说明没有安装过

安装 Node.js

安装 Node.js 的最佳方式是使用 nvm。

cURL:
$ curl https://raw.github.com/creationix/nvm/master/install.sh | sh
Wget:
$ wget -qO- https://raw.github.com/creationix/nvm/master/install.sh | sh
安装完成后，重启终端并执行下列命令即可安装 Node.js。
$ nvm install stable
或者您也可以下载 安装程序 来安装。

安装 Hexo

所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo。

$ npm install -g hexo-cli

安装完 Hexo 以后就可以开始制作自己的blog了

* 创建gitHub仓库
